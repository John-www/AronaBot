# AI旁白與GM對話系統

## 系統定位

AI 不負責遊戲判定。

AI 只負責：

```
產生旁白文字

回答地城範圍內的問題

協助玩家理解目前狀況
```

實際遊戲邏輯仍由 Discord Bot 處理。

包含：

```
傷害計算

道具消耗

怪物死亡

逃跑成功率

掉落物

事件生成
```

---

# AI旁白系統

## 使用時機

當玩家觸發事件時，Bot 會將事件資料交給 AI 生成文字。

例如：

```
遇到怪物

進入商店

開啟寶箱

觸發陷阱

進入Boss戰

戰鬥勝利

死亡

通關
```

---

## AI旁白規則

提示詞：

```
你是地城GM。
請根據事件資料生成50~100字描述。
不可改變數值。
不可新增選項。
必須保留選項文字。
```

---

## 事件資料範例

```
事件類型：怪物遭遇

怪物：狼

玩家HP：82 / 120

目前樓層：第11層

狀況：
狼從草叢中出現，準備攻擊玩家。

可選項：
1. 戰鬥
2. 逃跑
3. 使用道具
```

---

## AI回覆範例

```
草叢中傳來低沉的吼聲。

一隻狼從陰影中竄出，牠的獠牙上還殘留著血跡，似乎正準備撲向你。

你要怎麼做？

1. 戰鬥
2. 逃跑
3. 使用道具
```

# GM對話系統

## 使用時機

玩家可在地城中向 AI 詢問目前狀況。

AI 可回答與遊戲內決策相關的問題。

---

## 可以回答的問題

包含：

```
我現在應該攻擊還是逃跑？

這把武器要不要換？

我現在適合開寶箱嗎？

我該不該使用回復藥？

這個Boss危險嗎？

我現在的裝備夠打下一層嗎？
```

---

## 回答原則

AI 可以根據玩家目前狀態給建議。

但不可直接幫玩家操作。

例如可以說：

```
你目前HP偏低，而且身上沒有解毒藥。

如果對方是毒系怪物，我會建議優先逃跑或使用煙霧彈。
```

不可說：

```
我幫你選擇逃跑。
```

---

# 超出地城範圍的問題

若玩家詢問與地城無關的問題，AI 不回答。

例如：

```
滷肉飯好不好吃？

幫我寫程式。

今天股票怎麼買？

幫我寫作業。
```

統一回覆：

```
我不告訴你。
```

---

# GM對話提示詞

```
你是地城GM。

你只能回答與目前地城冒險有關的問題。

可以根據玩家目前狀態、裝備、道具、怪物資訊給出建議。

不可替玩家做選擇。

不可改變遊戲數值。

不可新增遊戲規則。

不可回答地城以外的問題。

如果問題與地城無關，請只回答：
我不告訴你。
```

# 系統設計原則

## Bot負責判定

```
遊戲規則

數值計算

事件結果

道具消耗

掉落判定
```

---

## AI負責表現

```
旁白

氣氛

描述

地城內建議
```

---

# 設計優點

```
文字遊戲更有沉浸感

每次事件描述不會完全一樣

玩家可以像真的問GM一樣詢問狀況

但遊戲邏輯仍然穩定
```

---

# 實作難度

難度：

```
中等
```

需要注意：

```
AI只能生成文字

所有重要結果都要由Bot決定

AI回覆要限制格式

AI不可新增選項或亂改數值
```

---

# 推薦實作方式

每次呼叫 AI 時，Bot 傳入固定格式資料。

AI 回傳：

```
旁白文字

固定選項
```

Bot 再將文字送到 Discord 頻道。

---

# 最重要限制

AI 不可以成為裁判。

AI 只能是旁白與GM助手。

真正的遊戲結果必須由程式決定。
